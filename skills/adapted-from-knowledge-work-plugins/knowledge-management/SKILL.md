# Knowledge Management (OpenClaw Adapted)

Organize and maintain support knowledge base. Create articles, FAQs, and troubleshooting guides.

## When to Use
- Creating knowledge base articles
- Organizing support documentation
- Writing troubleshooting guides
- Maintaining FAQs
- Structuring internal wikis

```markdown

**Category:** [Setup / Troubleshooting / How-To / FAQ]
**Last Updated:** [YYYY-MM-DD]
**Applies To:** [Product version / Plan level]

---

---

## Tags

`[tag1]` `[tag2]` `[tag3]`
```

## Article Types

### How-To Article
**Purpose:** Teach users how to accomplish a task
**Structure:** Goal → Prerequisites → Steps → Verification

### Troubleshooting Guide
**Purpose:** Help users resolve specific errors or issues
**Structure:** Symptoms → Causes → Solutions → Prevention

### FAQ
**Purpose:** Answer common questions
**Structure:** Question → Direct answer → Additional context

### Reference Documentation
**Purpose:** Provide detailed technical information
**Structure:** Overview → Parameters → Examples → Notes

## File Storage

KB articles saved to:
- `/memory/knowledge-base/[category]/[article-name].md`
